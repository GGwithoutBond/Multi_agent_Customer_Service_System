# 毕业设计：基于多智能体协作的智能客服系统

## 1. 主要研究内容

随着大语言模型（Large Language Models, LLM）、多智能体系统（Multi-Agent System, MAS）以及检索增强生成（Retrieval-Augmented Generation, RAG）技术的快速发展，智能客服系统正从传统的“单轮问答工具”向“具备任务理解、流程协同与知识增强能力的智能服务平台”演进。相较于传统单智能体客服模式，多智能体架构能够通过任务拆分、角色分工和协同调度，将复杂客服请求分发给不同的专用智能体分别处理，从而在业务适配性、响应准确性、流程稳定性和系统可扩展性等方面展现出更明显的优势。与此同时，结合 GraphRAG、长短期记忆管理以及工作流编排机制，系统可以进一步提升对复杂业务规则、跨轮对话上下文和个性化服务需求的理解能力，推动智能客服从“规则驱动”向“认知驱动”和“协同驱动”的方向发展。

本研究旨在面向现代电商与在线服务场景，探索一种由多智能体协作机制驱动、以知识增强和上下文记忆为支撑的智能客服系统新架构。系统以 Orchestrator-Worker 模式为核心，通过 LangGraph 实现多智能体工作流编排，引入 GraphRAG 技术提升知识检索与多跳推理能力，并结合工作记忆、短期记忆和长期记忆机制，增强系统在多轮对话、业务办理、投诉升级、信息查询和个性化服务等场景中的综合处理能力。研究工作的目标不仅在于实现一个可运行的智能客服原型系统，还在于总结多智能体架构在客服场景中的关键设计方法与工程实践路径，为相关领域的研究和应用落地提供参考。

本选题的主要研究内容是设计并实现一个基于大语言模型、多智能体协作与 GraphRAG 的智能客服系统，重点围绕多智能体编排机制、专用 Worker 智能体设计、知识增强检索、对话记忆管理以及系统工程化实现等内容展开，具体研究内容如下：

1. **多智能体协同架构设计与任务编排机制研究**
   - 研究多智能体系统在客服场景中的组织方式，设计以 Orchestrator 为中心、多个专用 Worker 为执行单元的协同模式，使系统能够对用户请求进行识别、分流、执行和汇总。
   - 基于 LangGraph 构建工作流状态机，对用户输入后的意图识别、任务路由、节点执行、结果审查、失败回退和结果聚合等过程进行统一编排，提高系统处理复杂请求时的稳定性和可控性。
   - 结合用户情绪、请求紧急程度和业务类型，研究更加细粒度的路由策略，实现普通咨询、订单相关请求、商品咨询、投诉升级和人工转接之间的智能切换。
   - 探索多智能体架构在客服系统中的职责划分原则，分析中心调度与专用执行分离所带来的性能收益、可维护性提升以及后续扩展价值。

2. **面向客服业务的专用 Worker 智能体设计与实现**
   - 面向 FAQ 场景设计常见问题解答智能体，使其能够结合知识库检索结果给出更稳定、更符合业务语境的回答。
   - 面向订单场景设计订单 Worker，实现订单查询、物流跟踪、退货退款等典型业务的智能处理，并研究智能体与外部工具或业务接口的协同方式。
   - 面向商品咨询场景设计产品 Worker，使其能够结合商品数据、向量检索和知识库内容完成商品介绍、参数说明和推荐回复。
   - 面向负面情绪与投诉场景设计 Complaint Worker 与 Human Worker，研究在用户情绪激烈、问题复杂或风险较高时的升级策略，提升系统的人性化服务能力与风险处置能力。

3. **基于 GraphRAG 的知识增强检索机制研究**
   - 研究将非结构化文本知识与结构化图谱知识相结合的方法，利用 Milvus 向量数据库和 Neo4j 图数据库构建混合知识组织模式，增强系统对专业知识和复杂关系信息的表达能力。
   - 探索 GraphRAG 在智能客服中的应用机制，使系统不仅能基于语义相似度检索相关文本，还能基于实体、属性和关系完成更高质量的知识关联与多跳推理。
   - 研究知识构建流程，包括原始业务数据采集、文本切分、向量化处理、实体关系抽取、知识图谱组织和索引建立等关键步骤，为后续问答服务提供统一的数据支撑。
   - 对比传统 RAG 与 GraphRAG 在回答准确率、知识召回效果、逻辑一致性与幻觉抑制方面的表现，分析知识图谱增强策略对客服问答质量的提升作用。

4. **多层级上下文记忆与个性化服务机制研究**
   - 设计包含工作记忆、短期记忆和长期记忆的三级记忆架构，使系统能够同时处理当前任务信息、近期对话上下文和用户长期偏好特征。
   - 研究基于 Redis 的短期记忆管理方式，用于保存会话级历史记录、摘要信息和临时上下文，提高多轮对话中的连贯性。
   - 研究基于 PostgreSQL 和向量检索的长期记忆机制，结合用户画像、历史咨询记录和偏好信息，实现跨会话的个性化服务能力。
   - 探索记忆检索、上下文压缩、信息筛选与动态注入策略，降低长上下文带来的冗余干扰，使模型在保留关键信息的同时提升回答质量和效率。

5. **系统工程化实现、性能优化与综合评估研究**
   - 以后端 FastAPI 服务为基础，构建统一的接口层、业务服务层和智能体调度层，并结合 Celery、Redis、PostgreSQL 等组件实现异步处理与任务协同。
   - 以前端 Vue 3 + Pinia 为基础实现对话交互界面，支持实时消息推送、会话管理和用户操作反馈，提升系统的可用性与交互体验。
   - 利用 Docker Compose 搭建容器化运行环境，完成多服务组件的统一部署，为系统联调、功能验证和后续展示提供工程基础。
   - 通过单元测试、工作流测试、性能测试和场景化验证，对系统的功能正确性、并发处理能力、工作流执行稳定性和用户体验进行综合评估，分析系统存在的不足并提出改进方向。

## 2. 毕业论文（设计）工作进展情况（详述）

### 2.1 毕业设计目前进展

目前已基本完成多智能体客服系统原型设计与核心功能开发，实现了 Orchestrator 对 FAQ、订单、商品、投诉等 Worker 的路由调度，并初步具备多轮对话、知识检索和流式回复能力，多智能体协同效果已能在典型客服场景中体现。

### 2.2 与预期不符的任务主要为

与预期不符的任务主要是多智能体协同效果验证仍不充分，尤其在复杂意图识别、跨 Worker 结果衔接、长期记忆调用和异常场景下的稳定性方面，还需继续优化功能联动与整体响应质量。
